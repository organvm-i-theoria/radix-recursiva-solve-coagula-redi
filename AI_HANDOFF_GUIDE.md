# AI Handoff Guide

This document establishes a standard handoff pattern for future AI agents collaborating within the repository. The header and footer notation should wrap every new or updated document generated by an automated contributor to guarantee clear provenance.

## Header & Footer Standard
- **Header format**: `<<<AI-Handoff:BEGIN::Agent={AgentName}::Timestamp={ISO-8601}>>>`
- **Footer format**: `<<<AI-Handoff:END>>>`
- Include additional key/value pairs (e.g., `Task`, `Context`) when helpful for disambiguation.

## Implementation Notes
1. When generating or editing documents, insert the header on the first line and the footer on the last line.
2. Preserve existing headers/footers unless there is a strong reason to regenerate metadata.
3. For multi-agent collaboration, nest or sequence multiple blocks in chronological order.
4. Use UTC timestamps to keep chronology unambiguous across contributors.

## Example
```
<<<AI-Handoff:BEGIN::Agent=Gemini::Task=README_Update::Timestamp=2025-05-05T00:00:00Z>>>
# Sample Document
Content authored by Agent Gemini.
<<<AI-Handoff:END>>>
```
